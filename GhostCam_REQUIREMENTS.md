# プロジェクト概要: 定点観測カメラアプリ「GhostCam(仮)」

## 1. コンセプト
「昨日の状態を透かして、今日の変化を撮るだけ」
日々の微細な変化を同じ画角で定点観測するための超・最小構成カメラアプリ。

## 2. 技術スタック
- **Framework:** React Native (Expo) - クロスプラットフォーム(iOS/Android)対応
- **Camera:** `expo-camera`
- **Storage:** `expo-file-system` (ローカル保存のみ)
- **UI:** プレーンなReact Nativeコンポーネント (複雑なアニメーションや装飾は不要)

## 3. 画面構成と機能要件 (2画面のみ)

### Screen A: カメラ画面 (初期画面)
- **振る舞い:** アプリ起動後、即座に全画面のカメラプレビューを表示する。
- **ゴースト機能 (最重要):** - 前回の撮影画像が存在する場合、カメラプレビューの最前面に「透明度30% (opacity: 0.3)」でオーバーレイ表示する。
  - 画像はプレビューと完全に同じアスペクト比・サイズで重なること。
- **UI要素:**
  - 画面下部中央: シャッターボタン
  - 画面下部右側: ギャラリー画面への遷移ボタン (最新のサムネイルを表示)
- **撮影アクション:**
  - シャッターを切ると、画像をローカルのドキュメントディレクトリに保存する。
  - 保存完了後、その最新画像をただちに「次回のゴースト用画像」として状態（State）を更新する。

### Screen B: ギャラリー画面
- **振る舞い:** 保存された写真の一覧を時系列（新しい順）で表示する。
- **UI要素:**
  - 3列のグリッド表示 (`FlatList` 等を使用)。
  - 画面左上: カメラ画面に戻る「戻る」ボタン。

## 4. 制約事項（あえて実装しないこと）
以下の機能は今回のMVP（Minimum Viable Product）では絶対に実装しないでください。
- ユーザー登録、ログイン、クラウド保存機能（完全にオフラインで動作させる）。
- 写真の編集、トリミング、フィルター機能。
- アルバム分割、カテゴリ分け機能。
- 複雑な設定画面、チュートリアル画面。

## 5. 開発とリリースの要件 (Greenlight対応)
- App Store / Google Play ストアの審査を通過できるよう、カメラアクセス権限の記述（iOSの `Info.plist` 相当の `app.json` への `NSCameraUsageDescription` などの明記）を適切に行うこと。
